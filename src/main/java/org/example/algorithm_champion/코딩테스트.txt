좋은 습관
    1. 문제를 올바른 순서로 이해한다.
        - 읽기(시간, 메모리 제한, 문제 전체를 꼼꼼히)
            -> 이해하기(제공되는 정보(변수들) 정리, 예제 데이터에 대해 이해)
            -> 파악하기(가능한 최대, 최소 정답에 맞는 데이터를 직접 생성, 키워드가 되는 단어)

    2. 시간과 공간 복잡도를 계산한다.
        - 짤 가치가 있는지 먼저 확인할 수 있다.
        - 컴퓨터는 대략 1초에 1억번 연산을 할 수 있다.

    3. 코드를 효율적으로 함수화하여 구현한다.

    4. 코딩 테스트에서 "부분 점수"를 챙긴다.
